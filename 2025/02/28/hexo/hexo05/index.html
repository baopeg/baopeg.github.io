Hexo博客部署指南（五）—— 文章写作与静态生成

一、创建新文章

  • 默认在source/_posts目录生成Markdown文件
  • 使用hexo new draft "标题"创建草稿
  • 发布草稿：hexo publish "标题"

二、Front-matter 配置

字段 说明 必填
title 文章标题 是
date 创建时间（ISO 8601格式） 是
categories 分类（最多两级） 否
tags 标签（多个用数组或逗号分隔） 否
updated 最后更新时间 否

三、Markdown写作规范

如果显示图片插入错误，请检查图片路径是否正确。改错参考错误解决

四、生成与部署

五、高级技巧

  1. 自定义模板：
    在scaffolds目录创建自定义模板
  2. 资源文件夹：
    在Front-matter添加resource: true启用独立资源目录
  3. 文章摘要：
    使用<!-- more -->分隔摘要和正文

六、常见问题

  1. 图片路径错误：
    • 使用绝对路径/images/example.jpg
    • 启用post_asset_folder插件
  2. Front-matter格式错误：
    • 确保冒号后保留空格，英文冒号和中文冒号不能混用
    • 正确使用缩进
  3. 生成后样式丢失：
    • 清除缓存：hexo clean
    • 重新生成：hexo g --force
